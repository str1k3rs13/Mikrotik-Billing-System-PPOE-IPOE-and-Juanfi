Telegram (proof-of-payment approvals & alerts)

      
      
      

        When a customer uploads a receipt on the pay page, you get a photo in Telegram with Approve / Reject buttons. Approving auto-reconnects them. You'll also get an alert whenever someone is disconnected for non-payment. No public URL needed — the panel polls Telegram.
      

      
SMS (texting customers)

      
      
      
      
      
      
      

      
Dongle needs a classic modem-mode stick (Huawei E3531/E303 in modem mode, ZTE MF, SIM800 USB). HiLink-mode dongles (the 192.168.8.1 kind) do not expose a port. Find the port in Device Manager → Ports (COM & LPT).

      

        Used to text a customer when you reply to their ticket, if their contact is a phone number. Email replies use your SMTP settings above and can include the attached photo.
      

      
AI Assistant

      
      
      
      

        Turn this on and paste your Anthropic API key to use the AI command box (type instructions in plain English) and the daily AI digest. Get a key at console.anthropic.com. The key is stored only on this computer. Leave the box unchecked to run the panel without AI.
      


      
Client Portal Sync

      
      
      
      
      
      

        Your public portal (on a VPS) only collects applications, help tickets, and payment notices. This panel pulls them with the secret token — your router is never exposed. Applications become job orders automatically. Keep the token private.
      

      
      


      
Reminders & daily report

      
      
      
      

        
        
      

      
A daily summary (collections, active/suspended, expiring, pending proofs & tickets) is sent to your Telegram at the report time. Telegram commands: /today, /pending, /expiring, /remind.

      
      
Data & tools

      

        
        
        
        
        
        
        
        
      

      
      
      
      
Backup downloads all billing data (plans, customers, invoices, payments, users, settings, tickets) as a JSON file. Restoring replaces current data with the backup — do it carefully.


      
Dry-run safety mode

      
      
When ON, any command that would change the router (add/remove/set firewall, proxy, leases, VLANs, profiles) is logged but not sent. Reads still work. Use this to see exactly what the panel would do before letting it touch your live router. Turn OFF to apply for real.

      

      


      
Data maintenance (storage)

      
See what's taking space and archive old logs. Cleanup saves old records to a file first, then removes them from the live database (keeping the last N months). Your customers, payments, and invoices are never auto-cleared.
